
============================================================
  ORDO DIGITIZATION — WORKFLOW AT A GLANCE
============================================================

  PREFILL      Once a year in October, the Maintainer clicks
            one button. The system pre-fills next year's
            Calendar Sheet from the rubric engine and
            clones last year's Prose Doc as a starting point.

  REVIEW    Editors work in familiar Google tools:
            a Calendar Sheet (per-day approval),
            a Prose Doc (front/back matter),
            and a shared Necrology Sheet.
            Only flagged days and edited sections need signoff.

  PUBLISH   Lead editor clicks "Publish." Automation merges
            engine output + editor overrides + prose,
            then produces four files: JSON, HTML, PDF, DOCX —
            all with an auto-built Table of Contents.
            Files are posted to a public GitHub Release.

  CONSUME   Public web calendar, downloadable PDF/DOCX,
            parish-specific overlays, and a JSON API —
            all served free from GitHub. No servers to run.
